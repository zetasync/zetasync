## Engineering & Architecture

### 工程设计（Engineering Design）

- **模块化 / Monorepo 思维**  
  - 按「领域 + 能力」拆分仓库结构（核心库 / 业务模块 / 公共工具）  
  - 通过包管理（pnpm / npm / workspace）复用底层能力  

- **工程效率与自动化**  
  - 使用 CLI / 脚手架统一项目初始化与规范  
  - 引入 Lint / Format / Git Hooks 保证团队提交质量  
  - CI 中集成自动测试、构建与基础质量检查  

### 代码架构（Code Architecture）

- **分层架构**  
  - UI 层（View）✚ 状态层（Store / Hooks）✚ 业务层（Service）✚ 基础设施层（Infra）  
  - 清晰区分「业务逻辑」与「技术细节」（请求、缓存、日志等）  

- **可维护性 & 可测试性**  
  - 使用 TypeScript / 类型标注管理领域模型  
  - 提倡小而纯的函数、可组合的模块、可替换的实现  
  - 针对关键模块引入单元测试 / 集成测试  

- **面向演进的设计**  
  - 支持按业务拆分子系统、逐步服务化 / 微服务化  
  - 保持 API 与内部实现解耦，方便替换存储 / 算法 / 模型实现  

### 前端架构（Frontend Architecture）

- **组件化与设计体系**  
  - 使用组件库作为设计语言载体，统一交互与视觉规范  
  - 抽象基础组件（Button、Input、Layout）与业务组件（表单引擎、列表模板）  
  - 利用 tokens / 主题系统支持多品牌、多主题切换  

- **状态管理与数据流**  
  - 按页面复杂度选择合适模式（局部状态 / 全局 Store / Server State）  
  - 分离远程数据状态（Server State）和本地 UI 状态（UI State）  
  - 配合缓存策略与请求封装，减少耦合、降低心智负担  

- **性能与体验优化**  
  - 路由级拆分、按需加载、懒加载与骨架屏  
  - 利用虚拟列表、节流、防抖等手段优化大数据渲染  
  - 通过监控（Performance / UX 事件）持续观察和调优
